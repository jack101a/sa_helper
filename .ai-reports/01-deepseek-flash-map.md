## 1. Project Type and Framework

- **Type**: Full-stack monorepo for an AI-powered MCQ exam-solving assistant
- **Backend**: Python 3.11, FastAPI, Uvicorn
- **Frontend**: React 18, Vite 5, React Router 7, TanStack React Query 5
- **Extension**: Chrome/Firefox Manifest V3 browser extension
- **Database**: SQLite (primary, raw SQL via `sqlite3`) + SQLAlchemy ORM (new models, supports PostgreSQL)
- **AI/ML**: ONNX Runtime (text captcha OCR), Tesseract OCR, LiteLLM (LLM-based exam solving)
- **Testing**: pytest
- **Deployment**: Docker Compose, GitHub Actions (GHCR), Nginx, systemd

## 7. Auth / Session / User Logic

**Three-layer authentication** (from `backend/app/middleware/auth_middleware.py`):

1. **Admin session auth** (dashboard): Session cookie via `/admin/login` (username/password), validated by `_admin_guard()` in `admin_routes/utils.py`
2. **User-linked API keys** (new system): SQLAlchemy-based `UserApiKey` model, device binding, status checks (blocked/inactive/pending_payment)
3. **Legacy API keys** (existing system): SHA-256 hashed keys via `KeyService`, device binding, expiry/revocation checks

**Key files**:
- `backend/app/middleware/auth_middleware.py` — FastAPI middleware, dispatches both key types
- `backend/app/core/security.py` — Key generation, hashing, expiry helpers
- `backend/app/services/key_service.py` — Legacy key CRUD + validation
- `backend/app/services/user_key_service.py` — User-linked key CRUD + validation
- `backend/app/api/admin_routes/auth.py` — Admin login/logout
- `backend/app/api/admin_routes/utils.py` — `_admin_guard()` helper

**Constants**: `/v1/locators` and `/v1/field-mappings/routes` are public (no auth). Master key required for admin operations. `x-admin-token` header for key create/revoke.

## 8. State Management / Global App Logic

**Frontend** (React):
- **Server state**: TanStack React Query via `useQuery` in `useAdminData.js` — fetches `/admin/api/bootstrap` + 4 sub-queries, auto-refreshes every 30s
- **UI state**: `useState` in `App.jsx` — modal states, editing states, selection states, remembered keys (persisted to localStorage)
- **Theme**: React Context (`ThemeContext.jsx` + `useTheme.js`) — dark/light mode, persisted to localStorage
- **Toast notifications**: `useToast` hook in `App.jsx`
- **Handlers**: Custom hooks for key operations (`useKeyHandlers`), settings (`useSettingsHandlers`), models/mappings (`useModelHandlers`), proposals (`useProposalHandlers`)

**Backend** (FastAPI):
- **Dependency injection**: `Container` dataclass in `backend/app/core/container.py` — holds all initialized services, wired at startup via `build_container()`
- **Lifespan**: Startup starts solver service, sends startup alert, packages extension; shutdown stops solver, retrain service, exam service
- **App state**: `app.state.container`, `app.state.user_key_service`, `app.state.telegram_bot`
